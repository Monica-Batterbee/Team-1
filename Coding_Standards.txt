Naming
- Use clear, meaningful names.
- camelCase for variables and functions.
- PascalCase for classes.
- UPPER_CASE for constants.

Formatting
- Use consistent indentation (2 or 4 spaces).
- Keep lines under 100 characters.
- Remove unused code and imports.

Structure
- One file per component or module.
- Keep functions short and focused.
- Avoid repeating code (DRY principle).

Comments
- Explain “why” something is done, not “what.”
- Use TODO for unfinished work.

Git & Commits
- Commit often with short, clear messages.
- Use branches for features and fixes.

Reviews
- Be kind and constructive.
- Suggest improvements, don’t demand.

Testing
- Write tests for important logic.
- Keep tests simple and repeatable.
